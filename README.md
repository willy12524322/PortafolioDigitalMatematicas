## Sobre este portafolio

Este repositorio es una bitácora de aprendizaje estructurada de la asignatura **Matemáticas Discretas**. No se limita a recopilar resultados: documenta el *proceso* de construcción del pensamiento lógico-matemático, desde los fundamentos proposicionales hasta su aplicación en el diseño de algoritmos computacionales.

El objetivo central es desarrollar la capacidad de traducir problemas del lenguaje natural a modelos formales verificables, una competencia esencial para cualquier ingeniero de software que aspire a construir sistemas robustos y correctos.

> *"La lógica es el cimiento de toda computación. Entender sus leyes es entender cómo razona una máquina."*

## Unidades de aprendizaje

<details>
<summary><b>📘 Unidad 1 — Lógica Proposicional</b> &nbsp;·&nbsp; <i>Disponible</i></summary>

<br>

La primera unidad establece el lenguaje formal sobre el que se construye toda la asignatura. Se abordan los mecanismos que permiten representar el razonamiento humano como un sistema simbólico preciso y evaluable.

**Contenidos cubiertos:**

| Bloque | Temas |
|--------|-------|
| Fundamentos | Definición de proposición, proposiciones simples y compuestas |
| Conectores lógicos | Conjunción ∧, disyunción ∨, negación ¬, condicional →, bicondicional ↔ |
| Tablas de verdad | Construcción sistemática, clasificación de fórmulas |
| Leyes proposicionales | De Morgan, doble negación, conmutatividad, implicación material |
| Reglas de inferencia | Modus Ponens, Modus Tollens, Silogismo Hipotético |
| Aplicación computacional | Simplificación de condiciones lógicas en código |

<details>
<summary><b>🧭 Reflexión de formación profesional</b></summary>

<br>

Cursar Matemáticas Discretas supone un cambio de paradigma en la forma de programar. Antes de esta asignatura, escribir código significaba encadenar instrucciones de forma intuitiva; ahora significa construir una cadena de inferencias formalmente verificables.

Comprender que un condicional `if` en cualquier lenguaje no es más que una implicación lógica `p → q`, o que una expresión booleana compleja puede simplificarse aplicando las leyes de De Morgan, convierte al programador en un ingeniero que diseña con precisión. Esta base es indispensable para áreas como la verificación formal de software, la inteligencia artificial o la criptografía, donde un error lógico no es un *bug* menor, sino una vulnerabilidad crítica.

</details>
